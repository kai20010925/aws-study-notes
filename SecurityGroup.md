# Security Group

## 概要

Security Groupは、EC2などに対する通信を制御する仮想ファイアウォールです。

どのポートをどこから許可するかを設定し、必要な通信だけを通すために使います。

## 主な役割

- EC2へのインバウンド通信を制御する
- EC2から外部へのアウトバウンド通信を制御する
- ポート番号や送信元IPを指定して通信を許可する
- 不要な通信を閉じてセキュリティを高める

## SAAでよく出るポイント

- Security Groupはステートフルに動作する
- 許可ルールのみを設定し、明示的な拒否ルールは設定しない
- EC2単位で関連付けて利用する
- WebサーバーではHTTPやHTTPSを必要に応じて許可する
- SSHは接続元IPを絞ることが重要

## 学んだこと

Security Groupは、EC2を安全に公開するために必ず理解しておく必要があると感じました。

特に、SSHを全世界に開放しないことや、必要なポートだけを許可することが基本的なセキュリティ対策になると学びました。
